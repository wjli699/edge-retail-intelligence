## Project Overview

Edge Retail Intelligence Engine — an edge AI system for retail analytics (loitering detection, person tracking, cross-camera ReID). The architecture is a **decoupled hybrid stack**:

- **C++ core engine** (`core_engine/`): real-time GPU video pipeline (DeepStream/GStreamer/TensorRT), publishes JSON events over ZeroMQ
- **Message bus** (`messaging/zmq/`): ZeroMQ PUB/SUB decouples inference from business logic
- **FastAPI control plane** (`api/`, Phase 5+): multi-camera correlation, loitering logic, REST API
- **Optional**: PostgreSQL/TimescaleDB, vector DB for ReID embeddings, UI dashboard

Target hardware: NVIDIA Jetson Orin NX (JetPack 5.x, DeepStream 6.2, GStreamer 1.16.3).

## Build — C++ Core Engine

```bash
# First time or after dependency changes:
./scripts/install_deps_phase2.sh

cd core_engine
mkdir -p build && cd build
cmake ..
make -j$(nproc)
```

Override DeepStream path if non-standard: `cmake -DDS_SDK_PATH=/opt/nvidia/deepstream/deepstream-6.2 ..`

## First-time model setup

```bash
./scripts/download_peoplenet.sh   # NGC CLI or NGC_API_KEY required
```

TRT engine is auto-generated by `nvinfer` on first run (~5 min on Jetson Orin NX), then cached.

## Run

**Terminal 1 — C++ pipeline (publishes to ZMQ):**
```bash
cd core_engine
./build/edge_retail_core_engine --config configs/default.yaml --verbose
```

**Terminal 2 — Python consumer:**
```bash
cd messaging/zmq
pip3 install -r requirements.txt
python3 consumer.py --endpoint tcp://localhost:5555 --filter person
```

For local debugging without ZMQ, switch `output.mode: stdout` in `configs/default.yaml`.

## Code Architecture

### C++ namespace
All core engine code lives under `edge::retail::core`.

### Class responsibilities

| Class | File | Role |
|-------|------|------|
| `App` | `src/app.cpp` | Loads YAML config, selects output mode (zmq/stdout/file), launches `Pipeline` |
| `Pipeline` | `src/pipeline.cpp` | Builds and runs the GStreamer/DeepStream pipeline; owns the GMainLoop |
| `Publisher` | `src/publisher.cpp` | ZeroMQ PUB socket wrapper; called from the `EventCallback` |
| `FrameEvent` / `Detection` | `src/metadata.cpp` | Data types + compact JSON serializer |

### Output modes (`output.mode` in `default.yaml`)

| Mode | Behaviour |
|------|-----------|
| `zmq` | Binds a ZMQ PUB socket on `output.endpoint` (default `tcp://*:5555`) |
| `stdout` | Writes JSONL to stdout — useful for debugging |
| `file` | Appends JSONL to `output.file` |

### DeepStream pipeline topology

```
nvurisrcbin(s)
  --[pad-added:vsrc_0]--> nvstreammux.sink_0
nvstreammux --> nvinfer(PeopleNet) --> nvtracker(NvDCF) --> nvdsosd --> fakesink
                                             ^
                                       GST_PAD_PROBE_TYPE_BUFFER
                                       on_buffer_probe → EventCallback → Publisher.send()
```

- `nvurisrcbin` emits dynamic `vsrc_%u` pads; `on_pad_added` requests `sink_%u` from nvstreammux (GStreamer 1.16 API: `gst_element_get_request_pad`).
- The pad probe sits on the **nvtracker src pad** so every event carries a stable `tracking_id` (`UNTRACKED_OBJECT_ID = 0xFFFF…FFFF` means tracker hasn't assigned one yet).
- ZMQ `send()` uses `ZMQ_DONTWAIT` — frames are dropped (not blocked) if no consumer is connected or HWM is hit.

### Config files

| File | Purpose |
|------|---------|
| `configs/default.yaml` | Top-level: sources, model paths, output mode + ZMQ endpoint |
| `configs/config_infer_primary_peoplenet.txt` | nvinfer config for PeopleNet ONNX; paths relative to this file's directory |
| `configs/config_tracker_NvDCF.yml` | NvDCF tracker parameters for `libnvds_nvmultiobjecttracker.so` |
| `configs/peoplenet_labels.txt` | Class labels: person(0), bag(1), face(2) |
| `models/peoplenet/` | ONNX + TRT engine (gitignored — large binaries) |

### JSONL event format
One object per frame with ≥1 detection:
```json
{"event":"frame","ts_ms":1713000000000,"source_id":0,"frame":42,
 "detections":[{"class":0,"label":"person","conf":0.85,
                "bbox":{"l":100.0,"t":50.0,"w":80.0,"h":200.0},"tid":3}]}
```
