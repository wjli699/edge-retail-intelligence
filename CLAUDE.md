## Project Overview

Edge Retail Intelligence Engine — an edge AI system for retail analytics (loitering detection, person tracking, cross-camera ReID). The architecture is a **decoupled hybrid stack**:

- **C++ core engine** (`core_engine/`): real-time GPU video pipeline (DeepStream/GStreamer/TensorRT), emits JSON events to stdout or a message bus
- **Message bus** (Phase 2+): ZeroMQ (edge) or Kafka (scalable) decouples inference from business logic
- **FastAPI control plane** (`api/`, Phase 5+): multi-camera correlation, loitering logic, REST API
- **Optional**: PostgreSQL/TimescaleDB, vector DB for ReID embeddings, UI dashboard

Target hardware: NVIDIA Jetson Orin NX (JetPack 5.x, DeepStream 6.2, GStreamer 1.16.3).

## Build — C++ Core Engine

```bash
cd core_engine
mkdir -p build && cd build
cmake ..
make -j$(nproc)
```

To enable tests (no tests yet): `cmake -DCORE_ENGINE_BUILD_TESTS=ON ..`

Override DeepStream path if non-standard: `cmake -DDS_SDK_PATH=/opt/nvidia/deepstream/deepstream-6.2 ..`

## First-time model setup

```bash
# Download PeopleNet ONNX from NGC (needs NGC CLI or NGC_API_KEY env var)
./scripts/download_peoplenet.sh
```

The TRT engine (`resnet34_peoplenet_int8.engine`) is auto-generated by `nvinfer` on the first run and cached for subsequent runs.

## Run

```bash
# Run from core_engine/ so relative config paths resolve correctly
cd core_engine
./build/edge_retail_core_engine --config configs/default.yaml --verbose
```

Output is newline-delimited JSON on stdout:
```json
{"event":"frame","ts_ms":1713000000000,"source_id":0,"frame":42,"detections":[{"class":0,"label":"person","conf":0.8543,"bbox":{"l":100.0,"t":50.0,"w":80.0,"h":200.0},"tid":3}]}
```

To write to file instead: set `output.mode: file` and `output.file:` in `configs/default.yaml`.

## Code Architecture

### C++ namespace
All core engine code lives under `edge::retail::core`.

### Class responsibilities

| Class | File | Role |
|-------|------|------|
| `App` | `src/app.cpp` | Loads YAML config (yaml-cpp), owns output stream, launches `Pipeline` |
| `Pipeline` | `src/pipeline.cpp` | Builds and runs the GStreamer/DeepStream pipeline; owns the GMainLoop |
| `FrameEvent` / `Detection` | `src/metadata.cpp` | Data types + compact JSON serializer for JSONL output |

### DeepStream pipeline topology

```
nvurisrcbin(s)
  --[pad-added:vsrc_0]--> nvstreammux.sink_0
nvstreammux --> nvinfer(PeopleNet) --> nvtracker(NvDCF) --> nvdsosd --> fakesink
                                             ^
                                       GST_PAD_PROBE_TYPE_BUFFER
                                       (on_buffer_probe → serialize_frame_event)
```

- `nvurisrcbin` emits dynamic `vsrc_%u` pads; `on_pad_added` requests `sink_%u` from nvstreammux and links them (GStreamer 1.16 API: `gst_element_get_request_pad`).
- The pad probe sits on the **nvtracker src pad** so every event carries a stable `tracking_id` (UNTRACKED_OBJECT_ID = 0xFFFF…FFFF means no tracker assigned yet).

### Config files

| File | Purpose |
|------|---------|
| `configs/default.yaml` | Top-level: sources, model config paths, output mode |
| `configs/config_infer_primary_peoplenet.txt` | nvinfer config for PeopleNet ONNX; paths are relative to this file's directory |
| `configs/config_tracker_NvDCF.yml` | NvDCF tracker parameters; fed to `libnvds_nvmultiobjecttracker.so` |
| `configs/peoplenet_labels.txt` | Class labels: person(0), bag(1), face(2) |
| `models/peoplenet/` | ONNX + auto-generated TRT engine (gitignored — large binaries) |

### Output contract
JSONL to stdout (one JSON object per frame that has ≥1 detection). Fields: `event`, `ts_ms`, `source_id`, `frame`, `detections[]`. Downstream consumers (Phase 2+ message bus) parse this stream.
